3. 컨센서스 커버리지 분석

분석일: 2026-03-21

3.1 현황

구분 종목 수
KOSPI/KOSDAQ 전체 상장 ~2,500개
DART 재무제표 수집 대상 ~840개
컨센서스 수집 성공 299개
전체 대비 커버리지 약 12%
재무제표 보유 종목 대비 약 35%

3.2 커버리지가 낮은 원인

원인 A: 실제 애널리스트 커버리지 부재 (주요 원인, ~70%)

  • 한국 매매수 애널리스트는 KOSPI 200 + KOSDAQ 150 내외 대형/중형주에 집중
  • WiseReport/FnGuide 페이지는 전 종목에 존재하지만 소형주는 데이터 자체가 없음
  • collect_consensus.py 82~85행의 SKIP 조건: rating is None and target_price is None
  • 이는 파이프라인 문제가 아닌 시장 구조적 한계

원인 B: 파이프라인 INNER JOIN 필터 (부차적 원인, ~30%)

  • collect_consensus.py 38~45행: financial_statements와 INNER JOIN
  • 재무제표가 없는 종목은 컨센서스 수집 대상에서 원천 제외
  • 상장 초기 기업, DART 수집 누락 종목 등이 대상에서 빠짐

3.3 개선 가능성 평가

INNER JOIN 제약을 제거해도 획기적인 커버리지 향상은 어려움. 실제 애널리스트 커버리지는 시장 전체 기준 400~600개 수준으로 추정되며, 현재 299개는 이미 상당 부분을 커버하고 있음.

소폭 개선 방향: INNER JOIN → LEFT JOIN 또는 별도 전수 스캔으로 재무제표 없이도 컨센서스가 있는 종목(상장 초기, 스팩 등) 추가 수집 가능.

4. 재무제표 수집 설계 결함 분석

분석일: 2026-03-21 (최초) / 수정일: 2026-03-21 | 심각도: HIGH

4.1 실제 구조 (초기 분석 오류 수정)

⚠️ 초기 분석("15개만 갱신")은 오류. 실제 DB 상태 확인 및 코드 재분석 결과로 교체.

DAG는 두 task로 구성되며, 각 task의 처리 대상이 다르다:

[Task 1] sync_corp_codes()
  → get_watchlist_tickers(): watchlist + 하드코딩 15개 조회
  → company_dart_info에 UPSERT (기존 레코드 유지, 신규/변경만 반영)
  → 즉, 과거 수동 스크립트로 쌓인 레코드는 삭제되지 않음

[Task 2] collect_financial_statements()
  → SELECT * FROM company_dart_info WHERE corp_code IS NOT NULL
  → company_dart_info 전체를 처리 대상으로 삼음

결론: sync_corp_codes는 company_dart_info를 제한하지 않는다. 수동 스크립트로 채워진 2,661개 전체가 DAG 처리 대상이 된다.

4.2 실제 DB 상태 (2026-03-21 확인)

테이블 레코드 수 비고
company_dart_info 2,661개 수동 스크립트로 전체 채움

4.3 핵심 결함: DART API 호출량 초과

주간 DAG 예상 API 호출량:
  2,661개 × 4 보고서(Q1/Q2/Q3/Annual) × 5년 = 53,220건/주
  CFS 실패 시 OFS 재시도 포함 → 최대 106,440건/주

DART API 일일 한도: 10,000건

실행 소요 시간 (0.5초 딜레이 기준):
  2,661 × 4 × 5 × 0.5초 = 약 14.8시간
  → 주간 DAG 실행이 완료되기 전 다음 실행이 시작될 수 있음

4.4 발생 가능한 장애 시나리오

시나리오 내용
API 한도 소진 수집 중간에 DART 응답 차단 → 이후 종목 전량 누락
DAG 실행 시간 초과 14시간+ 실행 → timeout 또는 다음 실행과 충돌
불균등 갱신 시총 순 정렬 없이 처리 시 어떤 종목이 갱신되는지 불명확

4.5 DART 자체 이유로 데이터가 없는 케이스 (소수)

케이스 내용
SPAC 합병 전 사업 실체 없어 재무제표 미제출
리츠(REITs) 별도 보고 체계, DART 구조 상이
신규 상장 직후 보고서 제출 기간 미도래
관리종목/거래정지 active = false 필터로 제외됨

4.6 적용된 해결책 (2026-03-22 코드 수정 완료)

dag_financial_statements.py 수정 내용:

변경 1: 시총 순 정렬 + 30일 내 갱신 종목 스킵

-- 수정 전
SELECT ticker, corp_code, corp_name
FROM company_dart_info
WHERE corp_code IS NOT NULL

-- 수정 후
SELECT d.ticker, d.corp_code, d.corp_name
FROM company_dart_info d
LEFT JOIN stock_info s ON d.ticker = s.ticker
WHERE d.corp_code IS NOT NULL
  AND d.ticker NOT IN (
      SELECT DISTINCT ticker FROM financial_statements
      WHERE updated_at > NOW() - INTERVAL '30 days'
  )
ORDER BY s.market_value DESC NULLS LAST

→ 최근 갱신된 종목 재수집 방지, 시총 상위 종목 우선 처리

변경 2: 수집 기간 단축 (주간 DAG 기본값 변경)

years_to_fetch = context.get("years_to_fetch", 2)  # 5 → 2

→ 주간 자동 실행 시 최근 2년치만 갱신

변경 3: 수동 백필 DAG에 5년치 옵션 분리

# 05_kr_financial_stmt_manual DAG
op_kwargs={"years_to_fetch": 5}

→ 전체 히스토리 재수집 필요 시 수동 트리거

4.7 수정 후 예상 호출량

상태 호출량 실행시간
수정 전 (전체 2,661 × 4 × 5) 최대 106,440건/주 약 14.8시간
수정 후 첫 실행 (2,661 × 4 × 2) 최대 42,576건/주 약 5.9시간
수정 후 이후 실행 (30일 필터 효과) 대폭 감소 (신규 공시분만) 1~2시간 내외 추정

잔여 과제: 30일 필터 이후에도 2,661개 전체가 대상인 첫 실행은 여전히 한도 초과 가능성 있음. 근본 해결책(DART list API 기반 증분 수집)은 별도 검토 필요.

5. 핵심 공백 및 개선 권고

4.1 우선순위 매트릭스

우선순위 항목 영향도 구현 난이도
1순위 파생 지표 analytics 레이어 구축 상 하
2순위 미국 기업 펀더멘털 추가 상 하
3순위 공매도 + 기관 세부 수급 (KRX) 중 중
4순위 스크래핑 취약점 해소 (네이버) 중 중
5순위 KOSPI200 선물/옵션 OI, P/C ratio 중 중
6순위 NDF, 외환보유고 등 FX 심화 하 중

4.2 상세 권고

[공백 1] 미국 기업 펀더멘털 없음

  • 현황: NASDAQ/NYSE 종목은 시세만 있고 재무제표/밸류에이션/컨센서스 없음
  • 권고: yfinance Ticker.financials, Ticker.info (P/E, EV/EBITDA, analyst target) 활용
  • 대안: Simfin API, Macrotrends (무료 기본 제공)

[공백 2] 파생 지표 analytics 레이어 없음

  • 현황: 각 테이블 독립 적재, cross-table 분석 지표 없음
  • 권고: analytics_* 테이블 군 생성, 다음 지표 사전 계산
  • 컨센서스 괴리율: (현재가 - 목표주가) / 목표주가 * 100
  • 실적 서프라이즈: 실제 실적 vs 컨센서스 추정치
  • PBR-ROE 스코어 (밸류에이션 매트릭스)
  • 기술지표 복합 신호 (MA골든크로스 + RSI 조합 등)
  • 컨센서스 revision momentum (목표주가 변화 추이)

[공백 3] 수급 데이터 깊이 부족

  • 현황: 외국인/기관 순매수 합계만 수집
  • 누락: 연기금, 투신, 사모, 은행 등 기관 세부 분류
  • 권고: KRX 데이터포털 API (data.krx.co.kr) 연동
  • 중요성: 연기금 순매수는 중장기 방향성 신호로 신뢰도 높음

[공백 4] 공매도 데이터 없음

  • 현황: 공매도 잔고, 공매도 비율 미수집
  • 권고: KRX 공매도 잔고 일별 데이터 + 대차잔고 수집 DAG 추가
  • 참고: KRX 데이터포털 제공 (무료)

[공백 5] KOSPI200 선물/옵션 포지셔닝 없음

  • 현황: 파생상품 시장 데이터 전무
  • 권고: Put/Call ratio, 미결제약정(OI) 수집
  • 활용: 시장 방향성 판단, 공포/탐욕 지수 보완

[공백 6] FX 데이터 단방향

  • 현황: 환율 레벨만 있고 변동성/포지셔닝 없음
  • 누락: 환율 변동성, NDF(역외선물환), 외환보유고
  • 권고: 한국은행 ECOS API에 외환보유고 시계열 추가
